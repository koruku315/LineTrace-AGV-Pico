# LineTrace-AGV-Pico

Raspberry Pi Pico 2 W（RP2350）と FreeRTOS で実装した、ライントレース型の自律走行ロボット（AGV）です。
3つの反射型センサーで黒ラインを検出し、PID制御でモーターを駆動してコースを走行します。

制御部はスーパーループではなく **FreeRTOS のマルチタスク構成** で実装し、
電源は降圧モジュールによって **単三電池のみで動作する単一電源化** を実現しています。

## 主な特徴

- **FreeRTOS による2タスク構成**：ライン検出とモーター制御を独立したタスクに分離
- **タスク間通信に Queue を使用**：センサー値の「鮮度」を優先した設計
- **PID（PD）制御**によるライントレース
- **単一電源化**：降圧モジュール（MP1584EN）で 6V → 5V を生成し、Pico とモーターを1つの電池で駆動
- **レジスタ／ハードウェア寄りの実装**：PWM・GPIO を直接制御

## システム構成

### ハードウェア

| 部品 | 型番・仕様 |
|------|-----------|
| マイコン | Raspberry Pi Pico 2 W（RP2350, Arm Cortex-M33） |
| ラインセンサー | TCRT5000 反射型センサー × 3 |
| モータードライバ | DRV8833 |
| モーター | TT ギアモーター × 2（2輪駆動） |
| 車体 | 2WD シャーシキット |
| 降圧モジュール | MP1584EN（6V → 5V、可変） |
| 電源 | 単三電池 × 4（6V） |

### 配線図（概要）


                       単三電池 6V
                          │
                       [スイッチ]
                          │
                 ┌────────┴────────┐
                 │                 │
          DRV8833 VM        降圧モジュール IN+
          （モーター電源）          │
                          降圧モジュール OUT+（5V）
                                   │
                          Pico VSYS（Pin39）

  GND は全て共通（電池 −、DRV8833 GND、降圧 IN−/OUT−、Pico GND）


### GPIO 割り当て

| 機能 | GPIO |
|------|------|
| 左センサー | GP0 |
| 中央センサー | GP1 |
| 右センサー | GP2 |
| 左モーター（BIN1 / BIN2） | GP8 / GP9 |
| 右モーター（AIN1 / AIN2） | GP6 / GP7 |
| センサー電源 | 3V3（Pin36） |

> **補足：** 物理的なモーターの左右と配線が逆になっていたため、
> ソフトウェア側で左モーターを BIN 系、右モーターを AIN 系に割り当てて整合させています。

## ソフトウェア構成（FreeRTOS 2タスク）

制御を役割ごとに2つのタスクへ分割し、FreeRTOS のスケジューラで並行実行しています。

\`\`\`
[ライン検出タスク]  --Queue(error/上書き)-->  [モーター制御タスク]
 line_task                                     motor_task
 優先度: 2                                      優先度: 3（最高）
 センサー読取→誤差                               PID計算→PWM出力
\`\`\`

### 各タスクの役割

- **ライン検出タスク（line_task, 優先度2）**
  3つのセンサーを 5ms 周期で読み取り、ラインの位置を誤差値（error）に変換して Queue へ送信。

- **モーター制御タスク（motor_task, 優先度3）**
  Queue から誤差値を受け取り、PID（PD）計算で左右モーターの出力を決定して PWM を出力。
  即応性が必要なため、最も高い優先度を割り当てています。

### 設計上の工夫

- **Queue を「サイズ1 + 上書き（xQueueOverwrite）」に**
  ライントレースでは常に最新のセンサー値が重要で、古い値が溜まると制御が遅れます。
  Queue サイズを1にして上書き方式にすることで、モーター制御タスクが常に最新の誤差値で動作します。

- **モーター操作を1タスクに集約**
  モーターを操作するのは motor_task のみとし、複数タスクからの同時アクセス（競合状態）を防いでいます。

- **優先度設計**
  制御の遅れが即コースアウトにつながるモーター制御を最高優先度に設定しています。

## PID 制御について

3センサー構成のため、ラインの位置は5段階（左端・左寄り・中央・右寄り・右端）で判定し、
それぞれに誤差値（-2 〜 +2）を割り当てています。ラインを見失った場合は、
直前の方向に基づいて急旋回し、コースへの復帰を試みます。

制御は PD（比例 + 微分）で行っており、積分項（KI）は使用していません。
カーブでの曲がりすぎ（オーバーシュート）と直線での蛇行のバランスを取りながら、
KP・KD・基準速度を調整しています。

| パラメータ | 役割 |
|-----------|------|
| KP | 誤差に比例した補正の強さ |
| KD | 誤差の変化に対するブレーキ（振動・オーバーシュート抑制） |
| BASE_SPEED | 直進時の基準速度 |
| MIN_SPEED | カーブ内側輪の逆転量（鋭い旋回用） |

## ビルド方法

### 前提環境

- Raspberry Pi Pico SDK 2.2.0
- VS Code + Raspberry Pi Pico 拡張機能
- ターゲット：Pico 2 W（`PICO_BOARD=pico2_w`）

### FreeRTOS-Kernel の取得

本リポジトリには FreeRTOS-Kernel 本体は含めていません（容量・ライセンスの都合）。
プロジェクトのルートで、Raspberry Pi 公式フォークをクローンしてください
（RP2350 対応のため、メインラインではなく公式フォークを使用します）。

\`\`\`bash
git clone https://github.com/raspberrypi/FreeRTOS-Kernel.git
\`\`\`

### 設定ファイル

`config.h` には環境依存の設定が含まれるため `.gitignore` で除外しています。
必要に応じてご自身の環境に合わせて作成してください。

### ビルド・書き込み

VS Code の Pico 拡張機能から「Compile Project」でビルド、
「Run Project (USB)」で書き込みます。

## 工夫・苦労した点

- **RP2350 への FreeRTOS 移植**
  RP2350 はメインラインの FreeRTOS では対応が限られるため、Raspberry Pi 公式フォークを使用。
  設定ファイル（FreeRTOSConfig.h）のビルドエラーを一つずつ解消して動作させました。

- **スーパーループから RTOS への移行**
  当初は単一のループで実装していた制御を、役割ごとのタスクに分割し、
  Queue による安全なデータ受け渡しに作り替えました。

- **左右モーターの反転バグの解決**
  「指示と逆に曲がる」現象の原因が、物理配線とドライバ出力の対応のずれにあることを特定し、
  ソフトウェア側で割り当てを入れ替えて解決しました。

- **単一電源化（電源設計）**
  当初はモーター用電池とモバイルバッテリーの2電源構成でしたが、
  降圧モジュールで 6V から 5V を生成し、テスターで出力電圧を確認した上で
  Pico の VSYS に供給することで、単三電池のみでの動作を実現しました。
